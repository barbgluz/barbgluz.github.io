Escalonador de Processos

Projeto para a disciplina de Sistemas Operacionais (MATA58)

Código

As tecnologias utilizadas para o desenvolvimento do projeto foram:

  • HTML
  • CSS
  • JQuery (JavaScript)

Frontend

O frontend permite a inserção dos dados necessários, criação dinâmica de processos, validação dos dados e exibição do escalonamento em um gráfico de Gantt.

Classe Grant é responsável por construir e atualizar o gráfico de Gantt.

  • Inicializa o gráfico, criando uma tabela para o tempo de execução e os processos.
  • A função tick avança o tempo e atualiza o gráfico.
  • A função redraw reinicia o gráfico.

Backend: Classe Process

A classe Process contempla os processos e possui os seguintes atributos:

  • id: Identificador do processo
  • momento de chegada: Tempo em que o processo chega ao escalonador
  • duração: Tempo de execução do processo
  • tempo passado: Tempo já executado pelo processo
  • tempo de término: Momento em que o processo é finalizado

Além disso, possui os seguintes métodos:

  • tick: Incrementa o tempo passado
  • arrived: Indica que o processo chegou ao escalonador
  • finished: Indica que o processo foi finalizado

Backend: Classe Scheduler

A classe Scheduler contém os dados do escalonador de processos, incluindo os seguintes atributos:

  • algoritmo: Algoritmo de escalonamento utilizado
  • quantum: Tempo máximo de execução para processos no Round Robin
  • sobrecarga: Tempo de sobrecarga de chaveamento
  • clock: Relógio do sistema
  • quantum atual: Quantum atual em execução
  • inExecution: Processo que está sendo executado
  • inOverload: Flag indicando sobrecarga
  • jobQueue: Fila de processos
  • readyQueue: Fila de processos prontos
  • completedProcesses: Lista de processos completos

A classe também possui os seguintes métodos:

  • status: Retorna o status atual do escalonador
  • isOver: Indica se o escalonamento foi finalizado
  • enterProcess: Adiciona um processo na fila de processos
  • tick: Incrementa o clock do sistema
  • syncQueues: Sincroniza as filas de processos
  • addToReadyQueue: Adiciona um processo na fila de prontos
  • syncReadyQueue: Sincroniza a fila de prontos
  • syncInExecution: Sincroniza o processo em execução
  • getAverageTurnaroundTime: Retorna o tempo médio de turnaround do escalonamento
